현재 '다판다' 앱(index.html)을 매장 실무에 맞춰 완벽히 동작하도록 아래 모든 기능과 카카오톡/PDF 내보내기 기능을 포함하여 개편하고 코드를 업데이트해 주세요.

[1. 기존 요약 카드 제거 및 상단 '소분·마진 정산 계산기' 구현]
- 상단의 기존 5가지 요약 카드를 제거하고 '소분·마진 정산 계산기' 카드를 배치.
- 입력값: 구입원가(원), 총 판매금액(원), 총무게(kg), 소분 수량(개)
- 실시간 계산 결과 (소수점 1자리까지 표시):
  * 마진율(%) = ((판매금액 - 구입원가) / 판매금액) * 100
  * 개당 소분량 = (총무게 / 소분수량) -> g 및 kg 자동 환산
  * 개당 판매가 = (총 판매금액 / 소분수량)
  * 개당 원가 및 이익금(원)

[2. 카테고리 확장 및 품목 ON/OFF 관리]
- 카테고리 버튼: [전체], [과일], [야채], [생선], [건어물/기타]
- 기본 샘플 데이터 포함:
  * 生鮮(생선 10종): 고등어, 갈치, 조기, 오징어, 명태, 굴비, 바지락, 멍게, 전복, 문어
  * 건어물/기타(30종): 김, 북어, 멸치(세멸/중멸/대멸), 참기름, 들기름, 된장, 고추장, 간장, 당면, 건새우, 황태채, 오징어채, 쥐포, 다시마, 미역, 잣, 호두, 아몬드, 깨, 콩, 팥, 차조, 수수, 찹쌀, 보리, 건표고, 곶감, 대추, 밤, 건가지
- 각 품목 카드에 'ON/OFF' 스위치 추가: ON(활성화) 상태인 품목만 판매 목록 및 공유/정산 대상에 포함.
- 각 품목 카드 입력 항목: [품종, 총수량(단위포함), 구입원가, 판매금액, 총무게(kg), 소분수량]
- 입력 시 개당 소분량(g/kg) 및 개당 판매가가 실시간 자동 계산됨.

[3. 카카오톡 공유 및 PDF 내보내기 기능 추가]
- 메인 화면 상단 또는 하단에 [카카오톡 공유] 버튼과 [PDF 내보내기] 버튼 배치.
- 활성화(ON)된 품목만 아래와 같은 텍스트 형식으로 파싱하여 내보내기:
  "다판다 상품 판매 목록 [YYYY년 MM월 DD일] 기준

  [과일]
  - 사과(부사) 4kg / 총수량 10박스 / 소분할개수 10개 / 개당소분 400g / 판매가 1,500원

  [생선]
  - 고등어 2kg / 총수량 15마리 / 소분할개수 5개 / 개당소분 400g / 판매가 3,000원"
- [카카오톡 공유]: 모바일 `navigator.share` API 또는 클립보드 복사(Fallback)를 활용하여 카카오톡이나 문자메시지로 즉시 전송 가능하도록 구현.
- [PDF 내보내기]: 브라우저의 인쇄/PDF 저장 기능(`window.print()`) 또는 HTML2PDF 라이브러리를 활용하여 깔끔하게 A4 PDF 문서 형태로 출력/다운로드되도록 구현 (인쇄 시 버튼 및 입력폼 등 불필요한 UI 숨김 처리 `@media print`).

[4. 일일 마감 및 기간별(월/분기/년) 매입·매출 정산]
- 일일 마감 섹션: 날짜별 자동 계산된 매입총액 + 당일 수입 입력[카드대금, 현금매출, 입금매출] -> 오늘 총수입 자동 계산.
- 기간별 통계 [월별 / 분기별 / 연간]: 선택한 기간의 총 매입금액, 총 수입금액, 순수익을 집계하여 표/리포트 형태로 표시.
- 모든 데이터는 LocalStorage에 안전하게 자동 저장/복원되어야 함.
